# /plan-tomorrow — Gate 5: what's tomorrow?

Commit to tomorrow's one thing now, in the light of the real day, before the
excuses form. The plan is a durable artifact: in a fragmented life you want to
pick it up cold two days later and start building, not spend your best minutes
remembering where you were.

**Don't** let a deferred item drift onto tomorrow's list without a decision — that
just resets its dodge. **Don't** plan an ambitious tomorrow that ignores how today
actually went.

## Steps

1. **Resolve state** per `reference/the-deferral-engine.md` → State resolution.
2. Determine today's and tomorrow's dates. Read today's `## EOD Audit` (if it's
   missing, tell the user to run `/eod` first), `goals.md`, and `patterns.md`.
3. **Draft tomorrow** — one primary and up to two secondary, built from: today's
   misses, the highest-leverage open goal, and any 3+ deferral.
4. **Handle deferrals explicitly.** Any task that hit a pattern alert either
   becomes tomorrow's primary with the re-commit reason attached, or is cut. It
   does not get to reappear quietly as item four.
5. **Write** a `## Tomorrow ([tomorrow's date])` block to today's daily note, so
   tomorrow-you sees it at the start of the window.
6. If the state is a git repo (or vault), commit: `docs: write tomorrow's plan to [today]`.
7. Present the plan in one short block.

## The one rule

Set tomorrow while today is still honest. The deferred-but-important thing goes
first — the reason it keeps slipping is that it never wins a fair morning fight.
Give it the fight tonight.
